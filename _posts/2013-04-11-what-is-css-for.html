--- title: "What is CSS for?" layout: post ---

HTML is by default: responsive, vector, and highly readable on any device. Think about that for a second.

There is an abundance of CSS on the web that constricts the inherent awesomeness of As CSS authors, we have a responsibility to stop this. If you write CSS that removes any of these three features, you should rethink its purpose. What problem could be so important to solve that it's worth solving at the cost of universal access to your content & readability?

CSS is not here to make our lives as developers easy. It's here to enhance the usability of the web. It should never limit the usability of the web.

When we write CSS, we should keep these points in mind.
